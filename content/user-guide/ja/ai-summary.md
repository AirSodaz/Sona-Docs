# AI要約

「AI要約（AI Summary）」は、すでに文字起こしテキストが存在する状態で、その横に要約（短い振り返り）を作成したい場合に使用します。これは文字起こし本文を書き換えるものではなく、またエクスポート時に自動的に出力ファイルに含まれることもありません。

## 推奨対象

- 会議、講義、またはインタビューの要点をすばやく確認する
- 元の文字起こしテキストを変更することなく、別個の要約を作成・保存する
- 別の要約ツールに移行することなく、同一エディタ内で作業を完結させる

## 開始する前に

- [ライブ録音](guide:live-record)、[バッチインポート](guide:batch-import)、または[ワークスペース、プロジェクト、インボックス](guide:workspace-projects-and-inbox)から取得した文字起こしセグメントがすでに存在すること。
- 「Settings > LLM Service」で「Summary Model（要約モデル）」を設定する準備ができていること。

## 最初に要約モデルを割り当てる

1. 「Settings > LLM Service」を開きます。
2. 「Feature Models（機能モデル）」で、「Summary Model（要約モデル）」にモデルを設定します。
3. 「Provider Credentials（プロバイダー認証情報）」で、使用するプロバイダーを開き、必要な接続情報を入力します。
4. 選択したモデルが推論モードをサポートしている場合は、「Reasoning Mode」を有効にし、「Reasoning Level（推論レベル）」を選択します。
5. 「Test Connection（接続テスト）」をクリックします。
6. 「Summary Model」が利用可能になったら、メインワークスペースに戻ります。

## 要約の生成または更新

1. すでにセグメントが存在する文字起こしを開きます。
2. 文字起こし画面から要約エディタを開きます。
3. 次回の要約生成時に異なるプロンプトを使用したい場合は、「General（一般）」、「Meeting（会議）」、「Lecture（講義）」などのテンプレートを切り替えます。
4. 「Generate（生成）」をクリックして要約を作成します。作成済みの要約を置き換えたい場合は、後から「Regenerate（再生成）」をクリックします。
5. 表現を調整したい場合は、要約エディタで直接編集できます。エディタからフォーカスが外れると、Sonaは自動的に内容を保存します。
6. 他の場所で要約を再利用したい場合は、「Copy（コピー）」をクリックします。

## 要約の保存状態について

- 「AI Summary」は、元のテキストを変更することなく、文字起こしの横に最新の要約レコードを1つ保持します。
- 現在の要約は文字起こし本文とは別個に保存されますが、再利用する前に表現を調整したい場合は、エディタで直接手動編集できます。
- その後文字起こしが編集、校正、または再文字起こしされた場合、古い要約は表示され続けますが、手動で再生成されるまで「情報が古い可能性（Outdated）」を示す警告が表示されます。

## お役立ちメモ

- 「Summary Model」は、「Polish Model（校正モデル）」や「Translation Model（翻訳モデル）」とは個別に設定されます。1つのプロバイダーでこれらすべてを兼ねることも、それぞれ異なるプロバイダーに分けることも可能です。
- 「AI Summary」には、LLM対応의 プロバイダーとモデルが必要です。Google翻訳プロバイダーは要約機能には対応していません。
- 「AI Summary」がまだ完全に設定されていない状態でも、要約を手動で入力または修正することは可能です。
- テンプレートを変更すると、次回生成時の対象プロンプトまたはプロジェクトのデフォルト値が変更されます。テンプレートごとに別個の要約ファイルが作成されるわけではありません。
- 文字起こしの横に要約を作成するのではなく、文字起こしテキスト自体を書き換えたい場合は、[AI校正と翻訳](guide:ai-polish-and-translate)に進んでください。
- 次のステップがエクスポートである場合は、[エクスポートと設定](guide:export-and-settings)に戻ってください。
- 要約の出力内容はエクスポートされるファイルには含まれません。他の場所で使用したい場合は、パネルから直接コピーしてください。
