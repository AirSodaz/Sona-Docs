`AI 摘要` 适合在转录已经生成之后，用较短时间得到一份贴着原文存在的归纳。它不会改写原文，也不会自动并入导出文件。

## 适合这些场景

- 需要快速回看一次会议、课堂或访谈的主要内容
- 希望保留原始转录不变，只额外生成一份摘要
- 想在同一个编辑器里继续使用摘要，而不是切到别的工具整理

## 开始前

- 你已经有来自 [实时录音](guide:live-record)、[批量转录](guide:batch-import) 或 [工作区、项目与 Inbox](guide:workspace-projects-and-inbox) 的转录分段。
- 你准备先在 `设置 > LLM 服务` 中配置 `摘要模型`。

## 先完成摘要模型配置

1. 打开 `设置 > LLM 服务`。
2. 在 `功能模型绑定` 中为 `摘要模型` 指定模型。
3. 在 `Provider 凭据` 中展开要使用的 provider，填写所需连接信息。
4. 点击 `测试连接`。
5. 确认 `摘要模型` 已可用后，再返回主界面。

## 生成或更新当前摘要

1. 打开任意一个已经有分段内容的转录。
2. 从转录界面打开摘要编辑区。
3. 如果希望下一次生成使用不同提示词，可在 `通用`、`会议` 或 `课堂` 模板之间切换。
4. 点击 `生成` 创建当前摘要；如果之后想替换它，点击 `重新生成`。
5. 如果要微调措辞，可以直接编辑摘要文本。焦点离开编辑区后，Sona 会自动保存。
6. 如果要把摘要内容带到其他地方使用，点击 `复制`。

## 摘要会保留什么状态

- `AI 摘要` 会以“当前摘要记录”的形式保存在转录旁边，不会改写原文。
- 当前摘要内容会和原文分开保存，但如果你想微调措辞，仍然可以直接编辑它。
- 如果之后又编辑、润色或重新转录了原始文本，旧摘要会继续保留，但会显示已过期提示，直到你手动重新生成。

## 补充说明

- `摘要模型` 与 `润色模型`、`翻译模型` 是分开配置的。可以共用同一个 provider，也可以拆开。
- `AI 摘要` 需要真正的 LLM provider 与模型，不支持 `Google Translate` 系列 provider。
- 如果 `AI 摘要` 还没有完成 LLM 配置，你仍然可以先手动编写或修改摘要。
- 切换模板只会改变下一次生成所使用的模板，或项目默认模板；它不会为每个模板各自保存一份独立摘要。
- 如果你主要想改写正文而不是生成贴着原文存在的摘要，请继续看 [AI 润色与翻译](guide:ai-polish-and-translate)。
- 如果你下一步是导出成稿，请回到 [导出与设置](guide:export-and-settings)。
- 当前摘要结果不会并入导出文件；如需复用，请直接从摘要面板复制。
